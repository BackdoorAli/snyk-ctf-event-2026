# Security Policy

## Status

This repository is **documentation-only** and is **not an actively maintained tool**.

It exists to document preparation, workflow, screenshots, and lessons learned from participation in the **Snyk CTF event held on February 12, 2026**. It is not intended to function as a production application, security utility, exploit framework, or supported software product.

Because of that:

- no service-level guarantees are provided
- no regular patch cycle is promised
- no compatibility guarantees are provided
- no active feature development should be expected

## Supported Versions

There are currently no supported release versions in the traditional software-maintenance sense.

| Version | Supported |
|---------|-----------|
| main    | Best-effort documentation updates only |

## Responsible Disclosure

Although this repository is documentation-focused, responsible disclosure still applies.

If you believe you have identified a legitimate security issue related to the repository contents, please follow these guidelines:

1. **Do not publicly disclose the issue immediately.**
2. **Do not include sensitive material in a public issue or pull request.**
3. Provide a clear description of the concern, including:
   - the affected file or section
   - the nature of the issue
   - potential impact
   - recommended remediation, if known

## What to Report

Relevant reports may include:

- accidental inclusion of secrets, tokens, credentials, or local environment artifacts
- unsafe instructions that should be clarified or removed
- links or references that expose sensitive data
- files that should not have been committed
- documentation mistakes that could create unnecessary operational risk

## What Not to Report

The following generally do **not** fall under this policy:

- theoretical issues unrelated to the actual contents of the repository
- reports expecting active support for a software product
- feature requests framed as security reports
- issues that originate from third-party platforms, tools, or services outside this repository
- challenge content that was already part of an authorised CTF environment

## Disclosure Process

On a best-effort basis, valid reports may result in one or more of the following actions:

- removal of sensitive content
- clarification of documentation
- revision of repo structure or screenshots
- addition of warnings or scope notes
- archival of outdated material where appropriate

Response times are not guaranteed.

## Scope Reminder

This repository documents activity performed in an authorised capture-the-flag environment for educational and portfolio purposes. It should not be interpreted as a maintained offensive security tool or as an invitation to test third-party systems.

## Contact

Use GitHub's private security reporting features if enabled for the repository. If private reporting is not enabled, avoid posting sensitive details publicly and instead use a minimal issue requesting a secure contact path.

## Acknowledgements

Good-faith, responsible reports are appreciated. Stay safe, stay ethical, stay free :P
